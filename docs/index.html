About This Training

Background, what to expect, and what you will be able to do after.

The 1k1RG Database Curation and Loading Workshop is a two-day, hands-on technical training for IT personnel responsible for operating and maintaining the SNP-Seek rice genomics platform. SNP-Seek runs on a specialized stack — PostgreSQL with the Chado biological schema, and HDF5 for genotype matrices. This training closes the gap between server administration and the domain-specific knowledge required to independently curate, load, and verify genomic data without constant support from the development team.

Day 1

Schema, Ontologies & Data Loading

  • 1k1RG project overview and the SNP-Seek technology stack
  • Linux essentials and SQL fundamentals for database operations
  • Biological ontologies — controlled vocabularies and how cvterm types every record in Chado
  • Chado schema deep dive — feature, featureloc, stock, variantset, and the PostgreSQL/HDF5 boundary
  • Lab: install PostgreSQL, restore a Chado training database, explore with psql
  • Lab: load a reference genome into Chado using \copy and GFF-based loader scripts
  • Lab: bulk-load a full genomic dataset using uploadDataset.sh
Day 2

HDF5, VCF Pipeline & Open Lab

  • HDF5 — why the genotype matrix lives outside PostgreSQL and how SNP-Seek reads it
  • VCF format — file structure, INFO/FORMAT fields, and what genotype calls represent
  • Full two-step pipeline: VCF → genotype matrix → HDF5 (hands-on run)
  • Open lab — finish unfinished exercises, troubleshoot, and ask questions
  • Closing Q&A and handover

Learning Outcomes

  • Explain how Chado uses cvterm as the semantic type system across all core tables
  • Set up a PostgreSQL Chado database and restore a training dump independently
  • Navigate and query the Chado schema using psql meta-commands and SQL joins
  • Load a reference genome into Chado from a pre-built CSV or a GFF annotation file
  • Run uploadDataset.sh to bulk-load stock, SNP, and variant data into Chado
  • Explain the PostgreSQL/HDF5 boundary — what lives in the database and what does not
  • Execute the full VCF → genotype matrix → HDF5 pipeline and verify the output
  • Diagnose and recover from common loader failures (term-resolution errors, ID conflicts)

Prerequisites

  • Linux command-line comfort — navigation, file operations, running scripts
  • SQL fundamentals — SELECT, JOIN, INSERT, basic relational tables
  • SSH access to the training server (credentials distributed before the session)
