Bulk Data Loading via Script

🕐 3:45 – 4:30 🎤 — 🕑 45 min

Session Overview

A bulk loading script automates data ingestion into the PostgreSQL Chado schema. This lab walks through using uploadDataset.sh to load a training dataset, followed by a separate exercise loading a reference genome directly with \copy.

What the Script Does

  • Validates input files (sample list, SNP positions, HDF5 file)
  • Looks up or inserts CV terms, organism, and dbxref entries
  • Generates intermediate CSV files for each target table:
    • StockSample.csv
    • SampleVarietySet.csv
    • SnpFeature.csv
    • Variant_variantset.csv
    • Snp_FeatureLoc.csv
  • Updates sequence values to prevent ID conflicts
  • Performs efficient bulk copy using \copy

Part 1 — Using uploadDataset.sh

uploadDataset.sh handles the full load — organism lookup, CSV generation, and bulk copy — in one command. Download it from the Related Files section above.

Prerequisites

  • PostgreSQL client (psql) and database access credentials
  • Input files ready:
    • Sample list file (e.g., sample_list.txt)
    • SNP positions file (e.g., pos_500_per_chr.txt)
    • HDF5 file (e.g., test_file.h5)

Step 1: Make the Script Executable

bash
chmod +x uploadDataset.sh

Step 2: Run the Script

Make sure uploadDataset.sh and your input files are in the same directory, then run:

bash
./uploadDataset.sh sampleds sampleds sample_list.txt pos_500_per_chr.txt 'Japonica nipponbare' test_file.h5
Parameter Description
DB_NAMETarget database name
VARIANT_NAMEVariant set name
SAMPLE_FILEPath to list of stock/sample names
POS_FILEPath to SNP positions file
ORGANISM_NAMEOrganism (genus + variety)
H5_FILE_NAMEHDF5 filename
ⓘ Once the script finishes, continue with the Verify the Load section below.

Part 2 — Loading a Reference Genome with \copy

This exercise loads reference genome feature data directly into Chado using PostgreSQL's \copy command. Make sure you are in the directory containing the CSV file before connecting.

Step 1: Create a New Database

bash
sudo -u postgres createdb chado_training

Step 2: Connect to the Database

bash
sudo -u postgres psql -d chado_training

Step 3: Load the Reference Genome Features

Once connected, run the copy command:

psql
\copy feature FROM '11-feature-nipponbare.csv' WITH (FORMAT csv, HEADER true)
ⓘ Once the load completes, continue with the Verify the Load section below.

Verify the Load

Run these checks inside psql after loading — they apply whether you used Option A or Option B.

Check Row Counts

psql
SELECT COUNT(*) FROM stock_sample;
SELECT COUNT(*) FROM snp_feature;
SELECT COUNT(*) FROM variantset;

Expected Outcome

  • Stock, stock_sample, SNP, and variant data are fully inserted.
  • Relationships between samples and SNPs are properly linked.
  • Data is queryable using SQL joins and integrated into SNP-Seek.
